In web development, the action and method attributes of an HTML form define where data is sent and how it is transmitted.

The method attribute specifies how to send form-data (the form-data is sent to the page specified in the action attribute).
The form-data can be sent as URL variables (with method="get") or as HTTP post transaction (with method="post").

While standard HTML forms officially only support GET and POST,

modern APIs frequently use PATCH and DELETE.


Core HTTP Method
	GET: Used for retrieving data. Data is appended to the URL as a query string,
		making it bookmarkable but not secure for sensitive info.

		Appends form-data into the URL in name/value pairs
		The length of a URL is limited (about 3000 characters)
		Never use GET to send sensitive data! (will be visible in the URL)
		Useful for form submissions where a user wants to bookmark the result
		GET is better for non-secure data, like query strings in Google


	POST: Used for sending data to a server, typically to create a new resource.
		Data is sent in the request body, allowing for larger and more secure transmissions.
		
		Appends form-data inside the body of the HTTP request (data is not shown in URL)
		Has no size limitations
		Form submissions with POST cannot be bookmarked
	

	PATCH: Used for partial updates to an existing resource.
		For example, changing only a user's email without resending their entire profile.

	DELETE: Used to remove a specified resource from the server.
